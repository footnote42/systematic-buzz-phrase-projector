# Implementation Plan: Editorial Layer & Page Design

---

## Summary

Add the editorial layer that transforms the app from a bare instrument into a finished
product page. Concretely: load a headline font (Space Grotesk), restructure the page
layout into an interactive section + scrollable editorial section, create a static
`EditorialSection` component with Broughton origin story and author's note, and add
mode-aware micro-copy to the manual input field.

No new state, no new localStorage keys, no new interactive components. All changes are
layout, typography, and static prose.

## Implementation Phases

### Phase 1 — Font & layout foundation

**Goal**: Load Space Grotesk; restructure page layout from vertically-centred to
top-aligned interactive + scrollable editorial.

**Files**:
- `src/app/layout.tsx` — import `Space_Grotesk` from `next/font/google`; add
  `weight: ['400', '600']`, `subsets: ['latin']`, `display: 'swap'`,
  `variable: '--font-space-grotesk'`; extend `<html>` className to include both
  font variables
- `src/components/BuzzPhraseApp.tsx` — change outer wrapper from
  `<main className="flex flex-col flex-1 items-center justify-center ...">` to a
  `<div className="flex flex-col min-h-screen">` containing two children:
  1. `<main>` with the interactive area (self-contained flex column, centred,
     `min-h-screen` or equivalent top-padding so generate button stays above fold)
  2. Placeholder `<section>` for editorial (filled in Phase 2)

**Verification**: `npm run dev` — controls still work; page no longer vertically
centres oddly when window is tall; build passes.

---

### Phase 2 — EditorialSection component

**Goal**: Static prose component with Broughton origin story, canonical quote, author's
note, and attribution footer.

**Files**:
- `src/components/EditorialSection.tsx` — new file:
  ```
  - <section> with bg-gray-50 border-t and max-w-2xl mx-auto px-6 py-10
  - <h2> with font-[family-name:var(--font-space-grotesk)] for the section heading
  - <p> for origin story body
  - <blockquote> for canonical Broughton quote (styled with left border or italics)
  - <p> for author's note (AUTHOR_NOTE constant — placeholder for Wayne Ellis to fill)
  - <footer> or <p> for attribution line
  ```
- `src/components/BuzzPhraseApp.tsx` — replace placeholder `<section>` from Phase 1
  with `<EditorialSection />`

**Copy template** (implementation fills these in):
- Origin story: "[3–4 sentences naming Broughton, Time magazine, 1968, the matrix
  concept]"
- Quote: *"No-one will have the remotest idea of what you're talking about. But the
  important thing is that they're not going to admit it."* — Philip Broughton, 1968
- `AUTHOR_NOTE`: `"[PLACEHOLDER — Wayne Ellis to supply 1–2 sentences in first person about the Modern/Chaos extensions]"`
- Attribution: "Original concept by Philip Broughton, 1968."

**Verification**: Editorial section renders below interactive area. Skin switching
does not affect it. Reads correctly at 375px.

---

### Phase 3 — ManualInput micro-copy

**Goal**: Add mode-aware hint text below the manual input label.

**Files**:
- `src/components/ManualInput.tsx` — add `hint: string` to `ManualInputProps`;
  render `<p className="text-xs text-gray-500 max-w-[7rem]">{hint}</p>` below the
  `<label>`
- `src/components/BuzzPhraseApp.tsx` — add `MANUAL_HINT` constant (see data-model.md);
  pass `hint={MANUAL_HINT[mode]}` to `<ManualInput>`

**Verification**: In dev server — switch between Original, Modern, Chaos modes;
confirm hint text updates correctly below the input label.
