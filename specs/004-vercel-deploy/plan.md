## Implementation Phases

### Phase 1 — Vercel project setup & initial deployment (US1, P1)

**Goal**: App is live at a public `*.vercel.app` URL with all features functional.

**Steps**:
1. Create Vercel project — import GitHub repo `systematic-buzz-phrase-projector`
2. Verify framework auto-detection: Next.js, build command `npm run build`,
   output directory `.next`
3. Confirm no environment variables required (V1 is fully client-side)
4. Trigger first deploy; confirm build log shows zero TypeScript errors and
   First Load JS ≤ 150 kB
5. Smoke test on `*.vercel.app` URL

**Verification**: App loads at preview URL; `257` Original → "systemised logistical
projection"; localStorage persists across page reload.

---

### Phase 2 — Custom domain (US2, P2)

**Goal**: `https://buzz.waynetellis.com` is the canonical public URL with valid HTTPS.

**Steps**:
1. In Vercel project settings → Domains → add `buzz.waynetellis.com`
2. Note the CNAME target Vercel displays (`cname.vercel-dns.com`)
3. In DNS provider for `waynetellis.com`: add CNAME record
   `buzz` → `cname.vercel-dns.com` (TTL 300 for fast propagation)
4. Wait for Vercel dashboard to show domain as "Valid Configuration"
5. TLS certificate provisioned automatically by Vercel (Let's Encrypt, < 5 min)
6. Verify HTTPS loads cleanly; verify HTTP redirects to HTTPS automatically

**Verification**: `https://buzz.waynetellis.com` loads with no cert warning; smoke test
passes on custom domain.
