Feature Specifications3.1 Mode ControllerInput: UI Radio buttons or a Slider select component ("Original", "Modern", "Chaos").Behavior: * Selecting Original sets the active dataset pointer to indices 0-9 of the historical database.Selecting Modern sets the active dataset pointer to indices 0-9 of the modern database.Selecting Chaos combines both arrays natively. The input index engine dynamically scales from handling modulo-10 calculations to modulo-20 calculations.3.2 Generator Engine (Random vs. Manual)Randomizer Trigger: A click handler picks random integer indices $I_1, I_2, I_3$ based on the length of the active matrix mode.Manual Input Matcher: A 3-digit text field validation layer. If a user types 257 in Original Mode, the system must parse:Index 2 of Column 1 (systemised)   Index 5 of Column 2 (logistical)   Index 7 of Column 3 (projection)   Output: An array or object passed directly to the active UI display renderer: ["systemised", "logistical", "projection"].  3.3 Voting and Favourite StorageUpvote Handler: A toggle action that adds the current phrase string along with its unique ID (mode + index digits) to a localFavourites array.Persistence Layer: Syncs to browser window.localStorage on state change, enabling persistence between user sessions without demanding external database overhead.3.4 Display Themes (UI/UX Layer)Structural Container: A clean, centered block element inside the Next.js page layout.Animation Specs:Split-Flap Component: Uses standard HTML elements with CSS backface-visibility: hidden and absolute positioning to animate the top and bottom halves of individual letters during data changes.Slot Machine Component: Uses standard list items wrapper inside an overflow: hidden container. Target translation heights calculated via translateY dynamically based on the word array indices.4.
